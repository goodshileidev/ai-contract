# 通用平台基础功能需求 - 03-通用用户界面功能

## 1. 功能概述

本模块定义了在平台多个列表页面（如项目列表、模板列表、文档列表等）中应具备的通用、一致的用户界面（UI）和用户体验（UX）功能。

## 2. 功能需求

### 2.1 列表通用功能

- **REQ-FUNC-103.1**: **分页 (Pagination)**
  - 当任何列表的数据超过一页时（例如，每页显示20条记录），必须提供分页控件。
  - 分页控件应包括：上一页、下一页、首页、末页以及页码跳转功能。
  - 控件应清晰地显示当前页码和总页数。

- **REQ-FUNC-103.2**: **排序 (Sorting)**
  - 对于所有关键列（如项目名称、创建日期、状态、更新时间等），用户应能够点击列标题进行升序或降序排序。
  - 当前的排序列和排序方向应有明确的视觉指示（例如，箭头图标）。

- **REQ-FUNC-103.3**: **筛选 (Filtering)**
  - 系统应提供基于一个或多个关键字段的筛选功能。
  - 常见的筛选字段应包括：项目状态（进行中、已归档）、创建者、时间范围等。
  - 用户可以组合多个筛选条件进行联合筛选。
  - 已应用的筛选条件应清晰地展示出来，并允许用户方便地清除。

- **REQ-FUNC-103.4**: **通用搜索 (General Search)**
  - 在每个主要的列表页面顶部，应提供一个搜索框。
  - 用户可以通过输入关键词（如项目名称、文档标题的一部分）来快速查找匹配的条目。
  - 搜索应支持模糊匹配，并实时（或在用户停止输入后）更新列表结果。

### 2.2 通用交互功能

- **REQ-FUNC-103.5**: **加载状态 (Loading State)**
  - 当系统正在加载数据或执行一个耗时操作时，必须向用户显示一个清晰的加载指示器（例如，加载动画、骨架屏）。

- **REQ-FUNC-103.6**: **空状态 (Empty State)**
  - 当一个列表为空时（例如，没有任何项目），页面不应显示空白。
  - 必须显示一个友好的“空状态”提示，包含简短的说明和一个引导用户进行下一步操作的按钮（例如，“暂无项目，立即创建一个新项目”）。

- **REQ-FUNC-103.7**: **通知中心 (Notification Center)**
  - 系统应有一个统一的通知中心入口，通常位于页面右上角。
  - 未读通知的数量应以角标形式显示在通知中心图标上。
  - 通知中心应聚合所有类型的通知，如：任务提醒、审批通知、@提及、系统公告等。
  - 用户可以点击通知跳转到相关页面，并将通知标记为已读或删除。

## 3. 非功能性需求

- **一致性**: 所有列表页面的分页、排序、筛选和搜索功能在外观和行为上必须保持高度一致。
- **响应性**: UI交互（如排序、筛选）应立即响应，列表数据更新时间不应超过1秒。
- **可访问性**: 所有UI控件应符合基本的Web可访问性标准（WCAG）。
