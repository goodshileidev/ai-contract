# 通用平台基础功能需求 - 01-用户认证与管理

## 1. 功能概述

本模块定义了平台所有用户都需具备的基础认证和管理功能，是系统安全和个性化服务的基础。

## 2. 功能需求

### 2.1 用户注册

- **REQ-FUNC-101.1**: 系统必须支持新用户通过邮箱进行注册。
- **REQ-FUNC-101.2**: 注册时，用户需要提供昵称、邮箱地址和密码。
- **REQ-FUNC-101.3**: 密码设置需符合一定的安全策略（例如：最少8位，包含大小写字母和数字）。
- **REQ-FUNC-101.4**: 系统需在用户提交注册信息后，向注册邮箱发送一封验证邮件。
- **REQ-FUNC-101.5**: 用户必须点击验证邮件中的链接以激活账户，未激活的账户无法登录。

### 2.2 用户登录

- **REQ-FUNC-101.6**: 系统必须提供基于“邮箱+密码”的登录方式。
- **REQ-FUNC-101.7**: 系统应提供“记住我”功能，以便在一段时间内保持用户登录状态。
- **REQ-FUNC-101.8**: 登录时，系统需进行凭据验证，验证失败时应给出明确的错误提示（例如“邮箱或密码不正确”）。
- **REQ-FUNC-101.9**: 系统应具备防止暴力破解的机制，例如在连续登录失败5次后，临时锁定账户15分钟。
- **REQ-FUNC-101.10**: (可选) 系统可支持第三方账号（如钉钉、企业微信）扫码登录。

### 2.3 密码管理

- **REQ-FUNC-101.11**: 用户在忘记密码时，可以通过“忘记密码”功能进行重置。
- **REQ-FUNC-101.12**: 密码重置流程需通过向用户注册邮箱发送重置链接来完成，链接具有时效性（例如60分钟）。
- **REQ-FUNC-101.13**: 登录后，用户可以在个人设置中修改自己的登录密码，修改时需要输入旧密码进行验证。

### 2.4 个人资料管理

- **REQ-FUNC-101.14**: 用户登录后，应能查看和编辑自己的个人资料。
- **REQ-FUNC-101.15**: 可编辑的个人资料至少应包括：用户头像、昵称。
- **REQ-FUNC-101.16**: 用户的邮箱地址作为唯一标识，不允许用户自行修改。

### 2.5 用户会话管理

- **REQ-FUNC-101.17**: 用户成功登录后，系统应为其建立一个会话（Session）。
- **REQ-FUNC-101.18**: 用户可以主动点击“退出登录”按钮，以安全地终止当前会话。
- **REQ-FUNC-101.19**: 系统应设置合理的会话超时时间（例如，闲置30分钟后自动登出）。

## 3. 非功能性需求

- **安全性**: 所有密码和敏感信息在传输和存储时必须加密。
- **易用性**: 注册、登录、密码重置流程应清晰、简单、易于操作。
- **响应性**: 所有认证相关操作的服务器响应时间应在500ms以内。
