# AI标书智能创作平台 - 项目文档总览

## 📚 文档概述

本目录包含 AI标书智能创作平台（AIBidComposer）的所有项目文档，按照标准化的8个子目录组织，便于查找和管理。

## 📂 文档目录结构

### ✅ 标准目录说明

```
docs/
├── 01-指引/           # 项目指引、规范说明
├── 02-需求/           # 需求文档、用户故事
├── 03-架构/           # 系统架构、技术选型
├── 04-设计/           # 详细设计、数据模型
├── 05-实现/           # 实现说明、开发日志
├── 06-测试/           # 测试计划、测试报告
├── 07-交付/           # 部署文档、发布说明
└── 99-知识/           # 知识库、FAQ、最佳实践
```

## 🚀 快速开始

### 新成员入职阅读顺序

1. **第1天**: 阅读 `01-指引/CLAUDE.md` 和 `01-指引/README.md`
2. **第2-3天**: 阅读 `03-架构/00-架构设计总览.md`
3. **第4-5天**: 根据角色阅读相应的详细文档
   - 前端工程师: `03-架构/01-项目结构设计.md`（前端部分）
   - 后端工程师: `03-架构/01-项目结构设计.md`（后端部分）、`03-架构/02-数据库设计.md`
   - 全栈工程师: 所有架构文档
4. **第2周**: 实际开发，参考 `04-设计/` 和 `05-实现/` 文档

## 📝 文档规范

### 文档元信息要求

每个文档必须包含以下元信息：

```yaml
---
文档类型: [需求文档/设计文档/实现文档/测试文档/知识库文档]
需求编号: REQ-YYYY-MM-NNN
创建日期: YYYY-MM-DD
创建者: [AI模型名称 / 开发者姓名]
最后更新: YYYY-MM-DD
更新者: [AI模型名称 / 开发者姓名]
状态: [草稿/评审中/已批准/已实现]
---
```

### 修改历史要求

每个文档必须包含修改历史表：

```markdown
## 修改历史

| 日期 | 版本 | 修改者 | 修改内容概要 |
|------|------|--------|-------------|
| YYYY-MM-DD | 1.0 | 模型名称/姓名 | 初始创建 |
```

详细规范请参考: `01-指引/CLAUDE.md`

## 🤝 文档贡献

### 创建新文档

1. 确定文档类型，选择对应目录
2. 按照文档规范创建元信息头部
3. 编写文档内容
4. 添加修改历史表
5. 提交 Git 并注明文档类型

### 更新现有文档

1. 更新文档内容
2. 更新"最后更新"日期和"更新者"
3. 在修改历史表中添加记录
4. 提交 Git

### 文档审查

- **技术审查**: 技术负责人审查技术文档
- **产品审查**: 产品经理审查需求文档
