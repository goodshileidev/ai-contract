## 🔍 API设计规范

### API设计原则
- **RESTful风格**: 遵循REST架构风格
- **版本控制**: URI版本化 (/api/v1, /api/v2)
- **统一响应**: 统一的响应格式
- **错误处理**: 标准化的错误码和错误信息
- **文档完善**: OpenAPI 3.0规范，自动生成文档

### 统一响应格式

**成功响应**:
```json
{
  "success": true,
  "data": any,
  "message": "操作成功",
  "timestamp": "2025-11-27T10:00:00Z"
}
```

**错误响应**:
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述",
    "details": {}
  },
  "timestamp": "2025-11-27T10:00:00Z"
}
```

### 认证方式
- **认证**: JWT + OAuth2.0
- **授权**: RBAC (Role-Based Access Control)
