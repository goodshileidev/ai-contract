## 五、关键技术决策

### 5.1 编辑器选择

**决策**: 使用 Blacknode 作为核心编辑器
**理由**:
1. aidev3已验证其稳定性和功能完整性
2. 支持插件架构，易于扩展
3. 团队已有使用经验
4. 避免重新开发的时间成本

### 5.2 变量系统架构

**决策**: 完全复用三层变量架构
**映射关系**:
```
aidev3                    →  AIBidComposer
模板变量 (Template)        →  招标需求变量
变量映射 (Mapping)         →  需求-能力映射
实例值 (Instance)          →  项目实际数据
```

### 5.3 AI能力设计

**决策**: 保留7种基础能力 + 标书特化扩展
```
基础能力（复用）:
1. 变量提取 → 招标需求提取
2. 文本扩展 → 方案内容扩展
3. 文本摘要 → 执行摘要生成
4. 文本转换 → 格式规范化
5. 合规审查 → 标书合规检查
6. 阶段转换 → 商务标/技术标转换
7. 大纲规划 → 标书结构生成

扩展能力（新增）:
8. 评分要点分析
9. 竞争对手分析
10. 风险点识别
```
