# AIBidComposer - 服务启动验证指南 - 附录

### A. 完整验证命令集合

```bash
# 一键验证脚本
./scripts/verify-services.sh

# 手动验证命令合集
# Docker环境
docker info && docker-compose --version

# 启动服务
docker-compose up -d

# 检查容器状态
docker-compose ps

# 检查健康状态
docker-compose ps | grep "(healthy)"

# PostgreSQL
docker-compose exec postgres pg_isready -U postgres
docker-compose exec postgres psql -U postgres -d aibidcomposer -c "\dx"

# Elasticsearch
curl -u elastic:${ELASTICSEARCH_PASSWORD} http://localhost:9200/_cluster/health

# Redis
docker-compose exec redis redis-cli -a ${REDIS_PASSWORD} PING

# RabbitMQ
curl -u guest:guest http://localhost:15672/api/overview

# MinIO
curl http://localhost:9000/minio/health/live

# Java服务
curl http://localhost:8080/actuator/health

# Python AI服务
curl http://localhost:8001/health

# 前端
curl http://localhost:5173

# 查看所有日志
docker-compose logs --tail=50
```

### B. 监控命令

```bash
# 实时资源使用
docker stats

# 特定服务资源使用
docker stats aibidcomposer-backend-java aibidcomposer-backend-python

# 磁盘使用
docker system df

# 网络连接
docker network ls
docker network inspect aibidcomposer-network
```

### C. 清理命令

```bash
# 停止所有服务
docker-compose stop

# 停止并删除容器
docker-compose down

# 删除容器和数据卷（警告：会丢失所有数据）
docker-compose down -v

# 清理未使用的资源
docker system prune -a
```
